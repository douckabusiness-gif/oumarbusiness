# Methode de travail - Oumar Business

## But

Ce document fixe la maniere de travailler sur le projet pour eviter le mode
"feature infinie" et garder un rythme simple, clair et controle.

## Regle generale

Chaque demande importante suit toujours cette sequence :

1. mini-plan
2. execution ciblee
3. cloture simple
4. attente du prochain choix

## Mini-plan obligatoire

Avant chaque nouveau module ou grosse modification, il faut donner un mini-plan
avec ce format fixe :

- Titre
- But
- Changements concrets
- Pages / modules concernes
- Test simple
- Resultat visible attendu

Le mini-plan doit aussi dire clairement :

- ce qu'on change
- ce qu'on ne change pas

## Discipline pendant le travail

Pendant l'execution :

- rester sur le sujet demande uniquement
- ne pas ouvrir un autre chantier en parallele
- ne pas proposer une nouvelle grosse idee sans le signaler clairement
- ne pas melanger plusieurs objectifs dans une seule livraison

## Cloture obligatoire

Quand la tache est terminee, la cloture doit toujours contenir :

- ce qui est termine
- ou cela se trouve
- comment le tester
- ce qui reste eventuellement a corriger

Apres cette cloture, on ne part pas directement sur une autre fonctionnalite.

## Choix possibles apres chaque livraison

Apres une etape, on reste limite a trois suites possibles :

1. Corriger
2. Simplifier
3. Passer a la prochaine etape validee

## Ce qu'on evite

On evite :

- d'empiler trop de pages
- de melanger plusieurs idees en meme temps
- de proposer une nouvelle grosse fonctionnalite sans stabiliser l'existant

## Intention produit

Cette methode est choisie pour garder un projet :

- plus clair
- plus calme
- plus structure
- plus facile a relire
- plus facile a tester

## Application immediate

A partir de maintenant, chaque nouvelle demande doit suivre cette methode.
