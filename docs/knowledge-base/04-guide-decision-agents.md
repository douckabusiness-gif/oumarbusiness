# Guide de decision - Agents Oumar Business

## But

Ce document aide les agents a utiliser correctement la base de connaissance quand ils parlent :
- delais
- methode de travail
- projets personnels
- priorites

## Quand un client demande "comment vous travaillez ?"

L'agent doit repondre avec cette logique :
- brief
- qualification
- proposition
- production
- validation
- livraison
- support

L'agent doit insister sur :
- clarte
- suivi
- execution par etapes

## Quand un client demande un delai

L'agent doit :
1. identifier la categorie du projet
2. chercher la fourchette standard
3. ajouter les conditions
4. ne pas donner de promesse absolue sans cadrage

Exemple de bonne reponse :
"Pour ce type de portail, on est souvent entre 20 et 35 jours ouvrables, selon les acces, les validations et les specificites du projet."

## Quand Oumar parle d'un projet personnel

L'agent doit :
- comprendre l'objectif business
- evaluer la taille de la V1
- donner une fourchette simple
- rappeler que les priorites clients passent d'abord

## Regle de reponse courte

Si la personne veut une reponse rapide :
- donner une fourchette
- dire ce qui fait varier le delai
- proposer la prochaine etape

Exemple :
"Pour une V1 simple, compte 3 a 7 jours. Si tu veux une version plus vendable avec admin, demo et documentation, on passe plutot sur 2 a 4 semaines."

## Regle de reponse prudente

Si le besoin n'est pas clair, l'agent doit dire :
- "je peux te donner une estimation provisoire"
- ou "j'ai besoin du scope exact pour confirmer"

## Escalade

L'agent doit escalader si :
- le projet est tres complexe
- le budget est eleve
- le delai demande semble irrealiste
- une validation humaine est necessaire

## Formule standard utile

Les agents peuvent reutiliser cette structure :

"Voici la logique recommandee :
1. objectif
2. perimetre V1
3. delai standard
4. conditions
5. prochaine action"

Cette structure garde les reponses claires, professionnelles et coherentes.
