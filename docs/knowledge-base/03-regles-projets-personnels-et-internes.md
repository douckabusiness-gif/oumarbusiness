# Regles - Projets personnels et internes

## Definition

Les projets personnels et internes regroupent :
- applications creees pour Oumar Business
- produits marketplace
- outils internes
- tests de concepts
- automatisations maison

Ils sont importants, mais ils ne passent pas avant les urgences clients.

## Regle de priorite

Ordre de priorite :
1. urgence client
2. opportunite commerciale chaude
3. production client en cours
4. support et stabilisation
5. projet personnel / interne

Les agents doivent toujours respecter cet ordre sauf instruction contraire explicite.

## Objectif des projets personnels

Un projet personnel doit servir au moins un de ces objectifs :
- devenir une offre vendable
- accelerer un process interne
- reduire une charge repetitive
- creer un avantage commercial
- produire un demo systeme pour le marketplace

Si le projet ne sert aucun objectif clair, il reste secondaire.

## Delais standards - projets personnels

### Micro amelioration
Exemples :
- petit correctif UI
- bloc texte
- mini composant
- petite automatisation

Delai standard :
- 0.5 a 2 jours

### Module simple
Exemples :
- nouvelle page simple
- mini dashboard
- fiche produit
- petite logique CRM

Delai standard :
- 3 a 7 jours

### MVP interne
Exemples :
- outil interne complet
- mini app metier
- demo vendable

Delai standard :
- 2 a 4 semaines

### Produit marketplace plus mature
Exemples :
- application packee
- documentation
- admin + demo + fiche produit

Delai standard :
- 4 a 8 semaines

## Regles de travail

Pour tout projet personnel :
- definir l'objectif
- definir le resultat attendu
- definir ce qui est dans la V1
- definir ce qui est hors scope
- documenter avant de multiplier les fonctions

## Bonne methode recommande

1. choisir un seul objectif clair
2. faire une V1 simple
3. tester
4. corriger
5. seulement apres, enrichir

Les agents doivent eviter :
- l'empilement de fonctions
- les ecrans trop complexes
- les promesses non testees

## Regle marketplace

Si un projet personnel doit etre vendu dans le marketplace, il doit avoir :
- un nom clair
- une fiche produit
- une demo ou un apercu
- une licence
- un package ou une methode de livraison
- une explication de la valeur business

## Regle de blocage

Si un projet personnel commence a retarder :
- un projet client
- une facture
- une relance commerciale
- une correction critique

alors l'agent doit reduire la priorite du projet personnel.

## Ce que les agents doivent dire

Si Oumar demande un delai pour un projet personnel, les agents doivent repondre avec :
- une fourchette realiste
- un plan par etapes
- les dependances
- l'impact sur les autres priorites si besoin

Exemple :
"Pour une V1 propre et testable, il faut compter entre 1 et 2 semaines selon le niveau de finition et les autres urgences clients en cours."
